# Хранилище договоров — CLEAR

Хранилище промптов для анализа и работы с договорами, составленных по методу **CLEAR**.

## О методе CLEAR

**CLEAR** — метод составления запросов в рамках промпт-инжиниринга. Фреймворк используется для рефакторинга промпта в командный формат и отражает чек-лист качества.

### Основные параметры оптимизации

| Параметр | Описание |
|----------|----------|
| **Task** (задача) | Какое действие необходимо выполнять? |
| **Context** (контекст) | Условия, аудитория и ограничения |
| **Input** (ввод) | Какие данные обрабатываются? Какой ответ считается корректным? |
| **Output** (вывод) | Формат результата |

### Составляющие метода

- **Concise** (краткость) — цель фиксируется в 1–2 предложениях, без размытости
- **Logical** (логичность) — задаётся повторяемый порядок шагов, снижающий случайность
- **Explicit** (явность) — указывается точный формат вывода для интеграции в рабочий процесс, со схемой и примерами
- **Adaptive** (адаптивность) — один шаблон под разные режимы (драфт/продакшн/деплой) с разными требованиями к качеству
- **Reflective** (рефлексивность) — встроенный контроль качества и самокоррекция

## Структура репозитория

Промпты в хранилище соответствуют принципам CLEAR и готовы к использованию в различных сценариях работы с договорами.
